# Deep House Lyric Structuring

## Overview

Turn source lyrics into a professional deep-house arrangement without damaging the song's language. Preserve complete thought-units, identify setup vs payoff hooks, and make the final act resolve by returning to the strongest hook.

## When to Use

- Adapting existing lyrics into `[Intro]`, `[Verse]`, `[Build]`, `[Drop]`, `[Bridge]`, `[Outro]` sections.
- Creating Suno or ACE-Step lyrics for deep house, melodic house, club remix, or late-night dance versions.
- User says a version “almost works” but fails near the end, feels incomplete, or sounds weird.
- Lyrics are in Telugu, Hindi, Tamil, or another native script where broken phrase boundaries sound awkward.

Do not use when writing completely new lyrics from scratch unless the user asks to preserve the structure of source stanzas.

## Core Pattern

Analyze before restructuring:

1. Mark each stanza as one of: hook, setup, image, pre-chorus/build, confession, bridge, payoff.
2. Detect rhyme endings and musical landings. In Telugu Sahiba, `-వా` asks, `-దే` lands emotionally, and `-గా/-నే` can sustain a drop.
3. Keep full stanzas intact. Do not stop a thought at a dependent line such as “నువ్వు మాట్లాడే మాటల్లో” if its completion is “మునిగి తేలిపోతూ... తడబడిపోతున్నాను”.
4. Use setup lines before payoff lines. A line like “నీకసలర్థంఅవుతుంద” creates tension; it is weak if looped alone as a drop.
5. End with emotional bridge, then hook reprise. Do not end on a broken-heart bridge unless the user explicitly wants an unresolved outro.

## Deep House Map

| Section | Lyric Function | Rule |
|---|---|---|
| Opening Hook | Immediate identity | Use strongest short stanza intact |
| Hook Echo | Reframe hook or setup | Keep it brief; avoid overusing setup lines |
| Verse 1 | Image or premise | Preserve complete image stanza |
| Pre-Chorus | Tension build | Must finish the full thought-unit |
| Drop 1 | Payoff | Repeat complete hook or strongest hook fragments |
| Verse 2 | Story expansion | Combine intact setup, image, and confession stanzas |
| Drop 2 | Bigger payoff | Prefer complete hook/payoff over repeated setup-only line |
| Final Bridge | Vulnerability | Keep full confession/mercy stanza intact |
| Outro Chorus | Resolution | Return to the opening hook after the bridge |

## Stanza Integrity Checks

Before finalizing, ask:

- Does every Telugu/Hindi/Tamil phrase have its dependent completion?
- Did any section end on a grammatical setup instead of a payoff?
- Are repeated lines hook-worthy, or merely explanatory?
- Does Act 3 move from confession to bridge to hook reprise?
- Is the last sung stanza a strong identity hook unless unresolved ending was requested?

## Example: Fixing A Weak Final Act

Weak pattern:

```text
[Drop 2]
(సాహెబా నీకసలర్థంఅవుతుంద)
(సాహెబా నీకసలర్థంఅవుతుంద)

[Outro Chorus]
సాహెబా కొంచం జాలి చూపేవా
సున్నితమైనది నా మనసే
తట్టుకోలేదే ముక్కలవుతుందే
```

Why it fails: the drop loops a setup line, and the outro starts mid-bridge without its first line.

Better pattern:

```text
[Drop 2]
(సాహెబా కాస్త దగ్గరికొచ్చేవా)
(ఇంకా ఆటాడిస్తావా)
(గుండెల్లో ప్రళయం రేగిపోతుందే)

[Final Bridge]
ఈ మాటలన్నీ నేను ఎవరితోని చెప్పను
సాహెబా కొంచం జాలి చూపేవా
సున్నితమైనది నా మనసే
తట్టుకోలేదే ముక్కలవుతుందే

[Outro Chorus]
సాహెబా కాస్త దగ్గరికొచ్చేవా
ఇంకా ఆటాడిస్తావా
గుండెల్లో ప్రళయం రేగిపోతుందే
```

## Common Mistakes

| Mistake | Fix |
|---|---|
| Splitting a stanza for section variety | Keep the stanza whole; move the whole block instead |
| Looping a setup line as the drop | Use complete hook/payoff lines for drops |
| Ending on a bridge | Add final hook reprise after the bridge |
| Treating rhyme as only sound | Check semantic function: ask, build, confess, land |
| Adding new rhyming phrases without permission | Reuse original lyrics only unless user explicitly allows new writing |

## Output Guidance

When the user asks for analysis, return both diagnosis and revised structure. When they ask for files, create a new version rather than overwriting prior lyric attempts.
