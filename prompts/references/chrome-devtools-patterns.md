# Chrome DevTools MCP Patterns

Common patterns for browser automation using Chrome DevTools MCP across the pipeline.

## Tools Used

Steps 3 (Find Lyrics), 5 (Upload to Suno), and 7 (Fetch Cover Art) use Chrome DevTools MCP:

- `navigate_page` — Navigate to URLs or go back/forward
- `take_snapshot` — Capture the current page state (a11y tree)
- `evaluate_script` — Run JavaScript and return results
- `click` — Click elements by uid
- `fill` — Fill form inputs
- `upload_file` — Upload files through file inputs
- `wait_for` — Wait for specific text to appear

## Common Navigation Patterns

### Navigate to URL

```
Tool: navigate_page
Type: url
URL: https://example.com/path
```

### Navigate Back

```
Tool: navigate_page
Type: back
```

### Take Snapshot After Navigation

Always take a snapshot after navigating to verify the page loaded:

```
Tool: take_snapshot
```

## Search and Extraction Patterns

### Search Google

```

```

### Extract Page Title

```javascript
() => document.title
```

### Extract Meta Tag Content

```javascript
() => document.querySelector('meta[property="og:image"]')?.content
```

### Extract Text Content by Selector

```javascript
() => {
  const selectors = [
    '.lyrics', '.lyric', '.song-lyrics', '.lyrics-body',
    '[class*="lyric"]', '[class*="Lyric"]',
    'article', '.entry-content', '.post-content', 'main p'
  ];
  for (const sel of selectors) {
    const el = document.querySelector(sel);
    if (el && el.innerText.trim().length > 100) {
      return el.innerText.trim();
    }
  }
  return Array.from(document.querySelectorAll('p'))
    .map(p => p.innerText.trim())
    .filter(t => t.length > 20)
    .join('\n\n');
}
```

## Page Verification Patterns

### Check for Element Presence

Take a snapshot and check the output for specific element descriptions:
- Buttons: look for `button labeled "Text"`
- Text inputs: look for `textbox labeled "Label"` or `textbox with placeholder "..."`
- Links: look for `link "Text"`
- Images: look for `img "alt text"`

### Verify Page Loaded

Always verify the page loaded by checking for expected elements:

1. Navigate
2. Take snapshot
3. Check output for expected element text
4. If missing, wait and take another snapshot

## Common Element Selectors

When taking a snapshot, elements are identified by:
- **Name**: The accessible name (label, aria-label, or text content)
- **Role**: The element type (button, link, textbox, etc.)
- **uid**: A unique identifier generated by the snapshot

### Finding Elements in Snapshots

Look for patterns like:
```
[123] button labeled "Create Workspace"
[456] textbox labeled "Workspace Name"
[789] link "Download"
```

The number in brackets `[123]` is the uid to use in `click` or `fill` operations.

## Error Recovery

### Page Won't Load

1. Check network connection
2. Try reloading: `navigate_page` with same URL
3. Try with `ignoreCache: true`
4. If persistent, the URL may be invalid

### Element Not Found

1. Take another snapshot — the page may have updated
2. Check if the element text matches exactly (case-sensitive)
3. Try alternative text patterns (some sites use aria-labels vs visible text)

### JavaScript Execution Failed

1. Check the script syntax
2. Verify the page is fully loaded before running
3. Check for CSP (Content Security Policy) restrictions
4. Try simpler selectors first

## Performance Tips

- Always take snapshots after navigation or major state changes
- Use `wait_for` when expecting async content to load
- Batch operations when possible (fill multiple fields, then click submit)
- The a11y tree is more reliable than CSS selectors for automation
